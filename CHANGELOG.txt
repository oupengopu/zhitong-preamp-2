    - STROBE 低电平延时: 20μs→40μs (3.3V 输出建立需 ≥36μs)
    - STROBE 高电平延时: 20μs→40μs
    - 原因: 3.3V 供电下模拟摆幅小, 输出建立比 5V 慢

  ▸ 修正2: 噪声门限提升
    - NOISE_GATE: 3→8
    - 原因: 3.3V 供电底噪约 100-200mV, 对应 ADC ≈ 8-15/255
    - 效果: 静音时频谱不再跳动

  ▸ 修正3: NTC 温度 IIR 低通滤波
    - 新增 _ntc_smooth 变量, NTC_IIR_ALPHA=0.15f
    - 时间常数约 7s (@10s 读取间隔)
    - 原因: 中值滤波后仍有 ±0.3°C 抖动

  ▸ 修正4: 确认 3.3V 供电安全
    - MSGEQ7 工作电压 2.7~5.5V, 3.3V 在规格内
    - ESP32-S3 GPIO 无需分压, 直连安全

  ▸ 修正5: 零漂校准
    - 新增 _r_offset[7] / _l_offset[7] 数组
    - setup() 丢弃前3帧期间采集偏置平均值
    - read() 中减去偏置后再缩放
    - 原因: 每片 MSGEQ7 静态偏置不同, 减去后静音更干净
    - 启动时 ESP_LOGI 输出偏置值供调试

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
目标: 最大化 ESP32-S3 运行效率, 消除所有性能瓶颈

修改文件: 智能前级2.0.yaml + msgeq7.h + pga2311.h

  ── A. sdkconfig 硬件调优 (智能前级2.0.yaml) ──

    ▸ CPU 频率锁定 240MHz
      CONFIG_ESP32S3_DEFAULT_CPU_FREQ_240: "y"

    ▸ 缓存配置
      数据缓存: 32KB→64KB  (CONFIG_ESP32S3_DATA_CACHE_SIZE_64KB)
      指令缓存: 16KB→32KB  (CONFIG_ESP32S3_INSTRUCTION_CACHE_SIZE_32KB)
      缓存行:   32B→64B    (数据+指令, 提升 DMA/PSRAM 命中率)

    ▸ PSRAM 配置
      八线模式: CONFIG_SPIRAM_MODE_OCT
      80MHz:    CONFIG_SPIRAM_SPEED_80M
      指令取自 PSRAM: CONFIG_SPIRAM_FETCH_INSTRUCTIONS (释放 Flash 带宽)
      只读数据取自 PSRAM: CONFIG_SPIRAM_RODATA (字体 glyph 不抢 Flash)
      允许 malloc: CONFIG_SPIRAM_USE_MALLOC

    ▸ LVGL 渲染加速
      SIMD 指令: CONFIG_LV_USE_DRAW_SW_ASM
      硬件 FPU:  CONFIG_LV_DRAW_SW_ASM_ASSUME_VFP

  ── B. SPI 速度提升 (智能前级2.0.yaml) ──

    ▸ TFT 显示 SPI2_HOST: 40MHz→80MHz (ESP32-S3 SPI2 支持 DMA)
    ▸ 显示组件 data_rate: 40MHz→80MHz

  ── C. LVGL 缓冲区 (智能前级2.0.yaml) ──

    ▸ buffer_size: 25%→100% (全帧 428×142×2=121KB, PSRAM 容纳)
    ▸ 效果: 零撕裂渲染, 脏区单次传输

  ── D. 频段增益整数化 (msgeq7.h) ──

    ▸ float BAND_GAIN[7] → int BAND_GAIN_Q8[7]
      {230, 256, 282, 320, 371, 435, 512}
      等效增益: ×0.90, ×1.00, ×1.10, ×1.25, ×1.45, ×1.70, ×2.00
    ▸ 运算: (float)r_val * BAND_GAIN[i] → (r_val * BAND_GAIN_Q8[i]) >> 8
    ▸ 收益: 14 次 float 乘法 → 14 次 int 乘法+移位, 约 5x 加速

  ── E. 单锁快照更新 (智能前级2.0.yaml) ──

    ▸ 原来: 8 个 ESPHome interval action (14 次 get_combined/get_peak 加锁)
    ▸ 现在: 1 个 interval lambda
      - msgeq7::get_frame() 一次加锁拷贝 combined[7]+peak[7]+temperature
      - 直接 LVGL API: lv_bar_set_value(bar, val, LV_ANIM_OFF)
      - 值变化才更新: lv_bar_get_value() != val 时才 set
      - 峰值线位置: lv_obj_set_y() 直接设置

  ── F. PGA2311 SPI 时钟 ──

    ▸ 初版: 4MHz
    ▸ 性能迭代曾提至 10MHz (错误)
    ▸ 详见迭代4修正

【迭代4】PGA2311 SPI 致命超频修正 ★安全修正★
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
修改文件: pga2311.h

  ▸ 问题: SPI 时钟从 4MHz 优化到 10MHz, 超出芯片绝对上限
  ▸ 依据: TI PGA2311 Datasheet Page 6, Timing Requirements
          fSCLK 最大 6.25MHz (时钟周期最少 160ns)
  ▸ 危险: 超频 60% 会导致移位寄存器错位
          数据错位时, 微小音量可能被解码为 0xFF (+31.5dB)
          → 瞬间满功率输出, 可烧毁喇叭/永久损伤听力
  ▸ 修正: clock_speed_hz: 10000000 → 4000000
  ▸ 理由: 4MHz 下 16bit 传输仅 4μs, 相比 15ms fade 步进可忽略
          音量渐变瓶颈不在 SPI 速度, 安全第一
  ▸ 安全范围: ≤6.25MHz (建议 ≤6MHz 留余量)

【迭代5】音频检测自动输入切换
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
目标: 当检测到音频输入时自动切换到对应通道, 多路同时输入按优先级选择

修改文件: 智能前级2.0.yaml

  ▸ 新增全局变量 auto_input
    - 类型: bool, restore_value=true, 默认 true (开启)
    - 控制自动切换功能的启用/禁用

  ▸ 新增脚本 auto_switch_input
    - 触发条件: 任意音频传感器状态变化 (on_press / on_release)
    - 防抖: 500ms 延迟等待所有传感器稳定 (mode: restart 自动重启计时)
    - 优先级: CD(0) > DAC(1) > PC(2) > AUX(3)
    - 切换逻辑:
      1. 检查 auto_input 开关 / system_ready / switching_input / standby
      2. 扫描所有 4 路音频传感器, 取最高优先级的活跃通道
      3. 所有通道无信号 → 保持当前输入 (不切走)
      4. 目标通道与当前相同 → 跳过
      5. 目标通道不同 → 执行 switch_input 脚本 (含继电器切换/音量渐变)

  ▸ 修改 4 路音频检测传感器 (audio_cd/dac/pc/aux)
    - 新增 on_press → script.execute: auto_switch_input
    - 新增 on_release → script.execute: auto_switch_input
    - 保留原有滤波: delayed_on 100ms / delayed_off 500ms

  ▸ 新增 HA 模板开关 auto_input_switch
    - 名称: "自动输入切换"
    - 图标: mdi:swap-horizontal
    - restore_mode: RESTORE_DEFAULT_ON
    - turn_on → auto_input=true
    - turn_off → auto_input=false

  ▸ 修改状态栏 lbl_audio
    - 新增 A:/M: 前缀指示自动/手动模式
    - A:CD DA PC AX = 自动模式
    - M:CD DA PC AX = 手动模式
    - x 位置调整: 88→70 (为前缀腾出空间)

  场景示例:
    - 仅 CD 有信号 → 自动切到 CD
    - CD+DAC 同时有信号 → 切到 CD (优先级高)
    - 当前在 CD, CD 停止, DAC 有信号 → 自动切到 DAC
    - 所有输入无信号 → 保持当前通道
    - auto_input=false → 不自动切换, 仅显示音频状态

【已知注意事项】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1. LVGL 对象指针访问
     代码中使用 id(spec_0)->obj_ 访问 LVGL 对象
     如果编译失败, 可能需要改为 id(spec_0)->lv_obj 或其他成员名
     (取决于 ESPHome 版本)

  2. PSRAM 配置 — 强绑定八线 (Octal) PSRAM
     a) Octal 模式绑定 (CONFIG_SPIRAM_MODE_OCT)
        此固件的 sdkconfig 强制开启八线 PSRAM 模式。
        如果烧录到 Quad PSRAM（四线）的廉价 ESP32-S3 开发板上，
        PSRAM 初始化失败，将导致无限 Bootloop（启动死机）。
        移植到其他 S3 开发板前，务必确认 PSRAM 类型并调整此选项。

     b) 指令/只读数据取自 PSRAM (CONFIG_SPIRAM_FETCH_INSTRUCTIONS / CONFIG_SPIRAM_RODATA)
        要求固件大小不超过 PSRAM 可用空间，否则启动失败。
        如遇启动问题，先关闭这两个选项排查。

  3. MSGEQ7 零漂校准前提
     假设启动时无音频输入, 如果开机时有信号, 偏置值会偏高
     导致正常播放时频谱偏低 — 可通过 ESP_LOGI 输出验证

  4. NTC 温度范围限制
     使用 6dB 衰减 (0~2.5V), 0°C 时 NTC≈31kΩ → V≈2.5V
     低于 0°C 可能削顶, 室内前级场景可接受

  5. PGA2311 SPI 绝对不可超频
     fSCLK 最高 6.25MHz, 超频可导致数据错位 → 瞬间满输出
     这不是性能问题, 是安全隐患 (可烧毁喇叭/损伤听力)
     当前 4MHz 是安全值, 如需提速也不应超过 6MHz

  6. SPI 总线分离
     显示 SPI2_HOST (DMA, 80MHz) 与 PGA2311 SPI3_HOST (4MHz)
     两者独立, 无总线共享冲突

  7. lvgl_compat.h extern "C" 版本依赖
     lvgl_compat.h 使用 extern "C" 绕过 managed_component 的 opaque 限制，
     直接声明 LVGL C API 函数。这种做法深度依赖当前 LVGL 版本（v9.x）的函数签名。
     升级 ESPHome 大版本（如 2026.x → 2027.x）时，如果底层 LVGL 调整了函数签名
     （如添加 const 限定符或新增缺省参数），将在链接阶段报错（Linker Error）。
     修复方法：删除 extern "C" 块，改用标准 #include <lvgl.h> 方式。
     这是编译期安全失败，不会导致运行时异常。
